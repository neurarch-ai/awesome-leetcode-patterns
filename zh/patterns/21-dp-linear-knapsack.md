# 21 - DP I：线性与背包

> 中文版。English: [21-dp-linear-knapsack](../../patterns/21-dp-linear-knapsack.md)

> **问题形态：**「爬 n 级台阶有多少种方法？」「在不打劫相邻两户的前提下能抢到的
> 最多的钱。」「一个连续子数组的最大和。」「凑出某个金额所需的最少硬币数。」「这个
> 数组能否被分成两个和相等的部分？」凡是一个序列或一组物品上的最优解或计数解，能被
> 分解成同一个问题在更小前缀或更小容量上的形式，都属于这一类。

动态规划通过把重叠子问题只求解一次并复用答案来解决问题。这第一个 DP 文件覆盖两种
基础形态：序列上的一维线性 DP，以及带容量预算的物品上的背包 DP。在这里掌握配方，
接下来几个文件里的字符串和网格 DP 就都成了变体。

![一维 dp 数组，箭头从 dp[i-1] 和 dp[i-2] 指向 dp[i]](../../assets/dp-linear.svg)

*线性递推：dp[i] 由它身后紧邻的两个状态构建而成（打家劫舍、斐波那契、爬楼梯）。*

## 信号

出现以下情况时考虑 DP：

- **「最大」「最小」「有多少种方法」「是否可能」**，作用于一个序列或一个集合，且
  贪心选择显然不安全、暴力是指数级的。
- **最优子结构**：`n` 的最优答案由更小输入（`n-1`、`n-2` 或更小容量）的最优答案
  构建而成。如果你能把答案写成关于更小实例的递推式，DP 就适用。
- **重叠子问题**：朴素递归会多次重算同一个调用。斐波那契式的分叉
  `f(n) = f(n-1) + f(n-2)` 就是标志。
- **每个元素上的一个选择**（「取它或跳过它」），共享一个预算（一个目标和、一个重量
  容量）。那种「对着预算取或不取」正是背包。

如果各个选择互相独立且局部最优可被证明就是全局最优，那是
[贪心](25-greedy.md)，不是 DP。DP 是当一个局部更差的选择可能导向全局更优答案时你
才用的东西。

## 思路

每个 DP 都是四个决策。把这些钉牢，代码自己就写出来了：

1. **状态**：`dp[i]`（或 `dp[i][c]`）到底*表示*什么？把它钉在一句精确的中文上，
   例如「前 `i` 户房子能抢到的最多的钱」或「凑出金额 `c` 所需的最少硬币数」。模糊的
   状态定义是大多数 DP bug 的根源。
2. **转移**：一个状态如何由更小的状态构建？这就是递推式，它通常编码了一个选择：
   `dp[i] = max(skip, take)`。
3. **基例**：你可以直接填的最小状态，`dp[0]`、`dp[1]` 或 `dp[...][0]`。把这些弄错会
   使之后每个值都偏移。
4. **求值顺序**：按某种顺序填状态，使得每个转移读取的值都已经算好。对线性 DP 是从左
   到右；对背包，顺序还决定了 0-1 还是完全（见下文）。

回报：一个分叉成 `O(2^n)` 次调用的朴素递归，一旦每个状态只算一次并被缓存，就坍缩为
`O(n)` 或 `O(n * capacity)`。你用一张多项式大小的表换掉了指数级的重算。

## 模板

**一维线性 DP，打家劫舍（跳过对比选取）：**

```python
# Time: O(n), Space: O(1)
def rob(nums):
    prev2, prev1 = 0, 0                    # best up to i-2, i-1
    for x in nums:
        # state: best money considering houses so far
        # transition: skip this house (prev1) or take it (prev2 + x)
        prev2, prev1 = prev1, max(prev1, prev2 + x)
    return prev1
```

**Kadane，最大子数组（运行中的最优值要么延伸、要么重启）：**

```python
# Time: O(n), Space: O(1)
def max_subarray(nums):
    best = cur = nums[0]
    for x in nums[1:]:
        cur = max(x, cur + x)              # extend the run, or start fresh at x
        best = max(best, cur)
    return best
```

**0-1 背包 / 子集和（每个物品至多用一次）：**

```python
# Time: O(n * target), Space: O(target)
def can_partition_to(nums, target):
    # dp[c] = True if some subset sums to exactly c
    dp = [False] * (target + 1)
    dp[0] = True
    for x in nums:                         # outer loop: items
        for c in range(target, x - 1, -1): # inner loop DOWNWARD: 0-1, no reuse
            dp[c] = dp[c] or dp[c - x]
    return dp[target]
```

**完全背包，零钱兑换（每种硬币可重复使用，求最少数量）：**

```python
# Time: O(amount * len(coins)), Space: O(amount)
def coin_change(coins, amount):
    INF = float('inf')
    dp = [0] + [INF] * amount              # dp[c] = min coins to make c
    for c in range(1, amount + 1):
        for coin in coins:
            if coin <= c:
                dp[c] = min(dp[c], dp[c - coin] + 1)
    return dp[amount] if dp[amount] != INF else -1
```

背包最重要的单个细节：**对 0-1 从大到小遍历容量**（使每个物品至多用一次），而
**对完全从小到大遍历**（使一个物品能在同一遍中被再次选取）。

## 变体

- **爬楼梯 / 不同方法数。** 与斐波那契相同的递推式 `dp[i] = dp[i-1] + dp[i-2]`。
  唯一的内容是基例以及「一种方法」意味着什么；空间坍缩为两个滚动变量。
- **计数背包（零钱兑换 II、目标和）。** 把 `min`/`max` 换成求和：
  `dp[c] += dp[c - coin]`。对「数组合数」把物品循环放外层、容量放内层；对「数排列数」
  交换循环顺序。这个顺序是一个常见的 bug。
- **子集和 / 分割等和子集。** 总和必须为偶数；目标是 `total // 2`；然后就是布尔型的
  0-1 背包。目标和（分配 + 与 -）经过代数 `positives = (total + target) / 2` 之后是
  同一个问题。
- **打家劫舍 II（环形）。** 把线性打家劫舍跑两次，一次排除第一户、一次排除最后一户，
  取最大值。是套在基础递推之上的一个包装技巧。
- **乘积最大子数组。** 带转折的 Kadane：同时追踪运行中的最大值和最小值，因为一个
  负数会把两者对调。

## 记忆化到制表到空间优化的递进

几乎每个 DP 都能写成三种方式，知道这个递进能让你起步正确、收尾迅速：

1. **自顶向下记忆化。** 把递推式写成一个递归函数并缓存结果（`@functools.lru_cache`
   或一个 dict）。最接近自然递归，最容易写对，只计算可达的状态。
2. **自底向上制表。** 把递归变成一个按依赖顺序填表的循环。没有递归深度限制，且循环
   顺序把求值顺序显式表达出来。
3. **空间优化。** 如果 `dp[i]` 只读 `dp[i-1]`（和 `dp[i-2]`），就用几个滚动变量替换
   数组，把 `O(n)` 空间降到 `O(1)`。背包的行同样能坍缩为单个一维数组。

先用自顶向下打样把递推式弄对，然后重写成自底向上，如果面试官要求再压缩空间。不要
一上来就微优化空间；先弄出一张正确的表。

## 陷阱

- **模糊的状态定义。** 如果你无法用一句话说清 `dp[i]` 表示什么，转移就会写错。先把
  那句话写下来，再写递推式。
- **0-1 对比完全的循环方向。** 在 0-1 问题里从小到大遍历容量会重复使用物品（错误
  答案）；在完全问题里从大到小遍历则禁止重复使用。这是头号背包 bug。
- **计数中的组合对比排列。** 零钱兑换 II 数的是组合（物品循环在外层）。交换循环数的
  则是有序序列，是一个不同的数字。
- **错误或缺失的基例。** `dp[0]` 通常是空输入的答案（`0` 个硬币、和为零时为 `True`、
  凑出「什么都不凑」有 `1` 种方法）。一个错误的基例会让下游处处偏一个常数。
- **覆盖了你还需要的值。** 在空间优化的线性 DP 里，同时更新 `prev2, prev1`（元组
  赋值），使得新的 `prev1` 不会冲掉转移仍要读的那个 `prev2`。
- **贪心够用时却去找 DP。** 如果带局部规则的单遍扫描可被证明是最优的，DP 就是过度
  设计；反过来，不要在一个局部更差选择可能全局取胜的地方硬套贪心。

## 后续问题与相关模式

- 「现在它作用于两个序列」或「它是子序列，不是连续的」会推向
  [DP II：子序列与字符串](22-dp-strings.md)，那里状态多出一个索引。
- 「它是一个网格、或一个区间、或一个当作位掩码的子集」会推向
  [DP III：网格、区间、位掩码](23-dp-grids-intervals.md)。
- 「枚举出真正的那些子集，而不只是数它们或优化它们」会推回到
  [回溯](20-backtracking.md)；DP 就是那同一棵决策树经记忆化坍缩后的形式。
- 「这里一个局部贪心选择可被证明是最优的」会推向
  [贪心](25-greedy.md)，当交换论证成立时，它就是没有表的 DP。
- 「递推式有一种我能在其上搜索的单调结构」（如最长递增子序列）连接到
  [二分查找](07-binary-search.md)，它把一个 `O(n^2)` 的 DP 变成 `O(n log n)`。
