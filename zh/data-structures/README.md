# 数据结构

> 中文版。English: [data-structures](../../data-structures/README.md)

[各类模式](../patterns/README.md)本质上是算法：如何在数据中穿行，抵达答案。本节讲的是更底层的一层：这些算法赖以运行的数据结构，每个操作的开销，以及在什么场景下该选用哪一种。模式告诉你「用单调栈」；本节告诉你栈到底是什么，为什么 `append` 和 `pop` 是 O(1)，以及为什么你会基于 Python list 来构建它。

如果你熟悉各种模式，却在「dict 查找是 O(1) 还是 O(log n)」上犯迷糊，那这一节就是你该读的。想查原始的操作开销表，请把[复杂度速查表](../complexity.md)一并打开对照。

每个文件都遵循同样的结构：

1. **它是什么** - 用直白的话讲清这个结构及其不变量。
2. **操作与复杂度** - 你必须掌握的开销表。
3. **Python 实现** - 使用或构建它的地道写法。
4. **何时用（何时不用）** - 决策线索。
5. **权衡与陷阱** - 藏在某个不起眼调用背后的 O(n) 意外。
6. **相关模式** - 哪些模式建立在这个结构之上。

## 何时选哪种结构

这张识别表，从问题指向能让它变廉价的结构：

| 你需要... | 选用 | 原因 |
|---|---|---|
| 按下标访问、按顺序遍历 | 数组 / list | O(1) 下标，对缓存友好 |
| 在 O(1) 内判断「我见过这个吗」 | 哈希集合 | O(1) 平均成员判断 |
| 键映射到值、计数 | 哈希表 / Counter | O(1) 平均 get 和 set |
| 处理最近优先（撤销、配对） | 栈 | 栈顶 O(1) 入栈和出栈 |
| 处理先进先出（BFS、调度） | 队列 / 双端队列 | O(1) 入队和出队 |
| 在两端插入或删除 | 双端队列 | 两端都 O(1)，不像 list |
| 反复取出最小或最大值 | 堆 | O(1) 查看，O(log n) 弹出 |
| 保持数据有序且插入删除都快 | 平衡 BST 或[有序容器](10-sorted-container.md) | O(log n) 有序操作 |
| 在数据变动时问「第 k 小」或「区间内有多少个」 | [有序容器](10-sorted-container.md)（bisect、SortedList） | O(log n) 顺序统计 |
| 带更新的区间聚合（求和、最小值） | [树状数组或线段树](../patterns/29-segment-tree-fenwick.md) | O(log n) 查询和更新 |
| 按前缀搜索或自动补全 | 字典树 | O(前缀长度)，共享前缀 |
| 建模连接、寻找路径 | 图 | 边就是关系 |
| 随着边的加入追踪连通性 | 并查集 | 近乎 O(1) 的 union 和 find |
| 在中间插入或删除且不搬移 | 链表 | 已知节点时 O(1) 拼接 |

## 选对结构等于解决了一半问题

大多数「巧妙」的面试解法，不过是把对的结构套用到显而易见的算法上。「两数之和」就是哈希表。「有效的括号」就是栈。「合并 k 个链表」就是堆。「省份数量」就是并查集。当你卡住时，别去找什么技巧；问一句：哪种结构能让那个昂贵的操作变廉价，算法往往就自然浮现了。[速查表](../cheatsheet.md)把问题措辞映射到模式；这张表把它映射到底层的结构。
