# 动态规划：完整的子模式地图

> 中文版。English: [dp-patterns](../patterns/dp-patterns.md)

动态规划不是一个模式，它是一个家族，而「我懂 DP」通常意味着「我懂它三四种形态」。
这一页就是地图：每个值得识别的 DP 子模式、定义它的状态与转移，以及它在本仓库里被
深入讲解的地方。候选人在 DP 上挣扎，是因为他们把每道题都当作全新的；实际上大约有
一打反复出现的形态，说出形态就给了你递推。

三个核心文件承载模板：[DP I：线性和背包](21-dp-linear-knapsack.md)、
[DP II：子序列和字符串](22-dp-strings.md)，以及
[DP III：网格、区间、位掩码](23-dp-grids-intervals.md)。
[状态机 DP](30-state-machine-dp.md)文件覆盖股票一族。这一页凌驾于它们之上，作为
识别索引。

## 通用配方

每个 DP，无论形态如何，都是四个决定。如果你能把这些填好，就能写出来：

1. **状态。** `dp[...]` 是什么意思，用一句精确的中文说清？「凑出金额 c 的最少
   硬币数。」模糊的状态是每个卡住的 DP 的根源。
2. **转移。** 一个状态如何从更小的状态构建出来？这就是递推。
3. **基例。** 最小的那些状态，直接填入。
4. **顺序。** 求解状态的次序，使每个依赖都已就绪。记忆化时递归替你处理；填表时你
   自己选循环顺序。

下面的子模式，只是对「状态是什么、转移是什么」这个问题反复出现的答案。

## 子模式

### 1. 线性 DP（一个序列，一个维度）

- **状态：** `dp[i]` = 考虑前 `i` 个元素（或在 `i` 处结尾）的答案。
- **转移：** 从 `dp[i-1]`，也许还有 `dp[i-2]`。
- **例子：** 爬楼梯、打家劫舍、最大子数组和（Kadane）、解码方法、O(n^2) 的最长
  递增子序列。
- **讲解于：** [DP I](21-dp-linear-knapsack.md)。

### 2. 背包（物品对预算）

- **状态：** `dp[i][c]` = 用前 `i` 个物品、容量为 `c` 时的最优值。
- **转移：** 拿物品 `i`（`dp[i-1][c - w_i] + v_i`）或跳过它
  （`dp[i-1][c]`）。
- **变体：** 0/1 背包（每个物品一次）、完全背包（允许重复，凑硬币）、多重背包、
  子集和、等和子集划分、目标和。
- **判断标志：** 「对一个共享上限，每个物品拿或不拿。」
- **讲解于：** [DP I](21-dp-linear-knapsack.md)。

### 3. 双序列 DP（对齐两个字符串或数组）

- **状态：** `dp[i][j]` = 一个序列的前 `i` 个和另一个的前 `j` 个的答案。
- **转移：** 匹配（`dp[i-1][j-1]`）或从一侧跳过（`dp[i-1][j]`、
  `dp[i][j-1]`）。
- **例子：** 最长公共子序列、编辑距离、不同的子序列、交错字符串、正则表达式和
  通配符匹配。
- **判断标志：** 两个字符串，且前缀的答案能构建出整体的答案。
- **讲解于：** [DP II](22-dp-strings.md)。

### 4. 单序列上的子序列 DP

- **状态：** 在 `i` 处结尾的 `dp[i]`，常带一个次级扫描或一个二分加速。
- **例子：** 最长递增子序列（用耐心排序做到 O(n log n)）、最长等差子序列、最长
  递增子序列的数量。
- **讲解于：** [DP II](22-dp-strings.md)。

### 5. 网格 DP（二维棋盘上的路径和面积）

- **状态：** `dp[i][j]` = 到达或在格子 `(i, j)` 结尾的答案。
- **转移：** 从你被允许来自的邻居（通常是上和左）。
- **例子：** 不同路径、最小路径和、最大正方形、地下城游戏、摘樱桃（双智能体网格
  DP）。
- **讲解于：** [DP III](23-dp-grids-intervals.md)。

### 6. 区间 DP（靠分割点求解一个范围）

- **状态：** `dp[i][j]` = 子数组或子串 `[i, j]` 的答案。
- **转移：** 选择范围内最后处理的东西，或 `i` 和 `j` 之间的一个分割点 `k`；合并
  两个子区间。按区间长度递增来求解。
- **例子：** 戳气球、矩阵链乘法、合并石头的最低成本、最长回文子序列、移除盒子。
- **判断标志：** 一个范围的答案依赖于把它分割，且 n 很小（O(n^3) 可接受）。
- **讲解于：** [DP III](23-dp-grids-intervals.md)。

### 7. 位掩码 / 状态压缩 DP（子集作为整数）

- **状态：** `dp[mask]`，其中 `mask` 是至多约 20 个物品编码进比特的一个子集。
- **转移：** 从子集里加上或移除一个元素（一个比特）。
- **例子：** 旅行商、访问所有节点的最短路径、划分为 k 个相等子集、最小不兼容度、
  指派问题。
- **判断标志：** n <= 约 20，且天然的状态是「我用了哪些物品」。
- **讲解于：** [DP III](23-dp-grids-intervals.md)。

### 8. 状态机 DP（少数几个带转移的模式）

- **状态：** `dp[i][mode]`，其中 `mode` 是一个小的固定集合之一（持有、现金、
  冷冻期）。
- **例子：** 整个股票买卖一族、粉刷房子、删除并获得点数。
- **讲解于：** [状态机 DP](30-state-machine-dp.md)。

### 9. 树上 DP（在一棵有根树上聚合）

- **状态：** `dp[node][...]`，由孩子的 DP 值算出，通常用一次后序 DFS。
- **转移：** 合并孩子，常常每个节点有两种情形（包含该节点或不含）。
- **例子：** 打家劫舍 III、二叉树上的摄像头、树的直径、最大路径和、统计子树。
- **判断标志：** 输入是一棵树，且每个节点的答案需要它孩子的答案。这就是
  [树 DFS](12-tree-dfs.md)「从孩子往上返回一个值」的思路，以 DP 的面貌出现。

### 10. 数位 DP（统计具有某数位性质的数）

- **状态：** 数字中的位置、一个 `tight` 标志（我们是否仍被上限约束），以及任何
  携带的性质（至今的数位之和、上一位数字）。
- **例子：** 统计 <= N 且无重复数位的数、数位和能被 k 整除的数、统计一个范围内具有
  某性质的整数。
- **判断标志：** 「`[0, N]` 里（N 高达 10^18）有多少整数满足某数位条件。」你无法
  枚举，所以你逐位构建这个数。
- **注意：** 进阶；标准面试里罕见，竞赛轮次里常见。

### 11. 概率与期望 DP

- **状态：** 一个配置加上到达它的概率或期望值。
- **转移：** 用每步的概率给它加权。
- **例子：** 棋盘上的骑士概率、分汤、新 21 点、掷骰子模拟。
- **判断标志：** 问题问的是在随机移动上的一个概率或期望值。

## 如何使用这张地图

当一个问题闻着像 DP（在带重叠子问题的选择上做「最大/最小/计数/是否可能」）时，
别开始写代码。把它对到上面某个形态：

- 一个序列，一个运行答案：**线性**。
- 对一个预算拿或跳过：**背包**。
- 两个字符串：**双序列**。
- 一块二维棋盘：**网格**。
- 一个由中点分割的范围：**区间**。
- n <= 20 且「哪个子集」：**位掩码**。
- 少数几个模式：**状态机**。
- 一棵树：**树 DP**。

然后写出状态语句、转移和基例。形态把这三样都交到你手上。当两个形态似乎都合适时，
写你能最精确陈述其状态的那个；一个你能用一句清楚的话说出来的状态，就是一个你能
实现的状态。
