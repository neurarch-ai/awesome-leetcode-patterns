# 例题精解

> 中文版。English: [walkthroughs](../../walkthroughs/README.md)

[各类模式](../patterns/README.md)讲解的是技巧，而这些精解则展示了完整的思考过程。每一篇都选取一道广为人知的题目，用[六步框架](../framework/solving-framework.md)从头到尾把它解出来：厘清题意、手算一个例子、给出暴力解、找到瓶颈并点出对应模式、写出代码，最后测试与分析。这才是一场出色的面试真正听起来的样子，被完整地叙述出来。

读一篇模式文件是为了学会那件工具；读对应的精解则是为了亲眼看它如何在框架下被挑选并应用。价值就在第 1、3、4 步，也就是写代码之前的思考，而这恰恰是一份光秃秃的答案所掩盖掉的部分。

## 如何使用它们

1. **先在不看解答的情况下硬啃这道题。** 打开 LeetCode 题目，用框架认真尝试一次，之后再读精解。
2. **对比的是你的过程，而不只是你的答案。** 你厘清约束条件了吗？给出暴力解了吗？在写代码之前点出瓶颈了吗？你*过程*中的缺口才是这些精解存在的意义。
3. **留意模式交接的那一刻。** 每篇精解都会走到第 4 步，并明确说出「这段重复的工作就是模式 X 的信号」。把这个从瓶颈到模式的动作内化，正是[速查表](../cheatsheet.md)所要反复训练的全部技能。
